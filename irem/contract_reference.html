Mimari — Nasıl Çalışıyor?

MultiSigWallet ← TEK deploy edilen contract (Hardhat ile) ├── is ReentrancyGuard ← OpenZeppelin (reentrancy koruması) ├── is OwnerManager ← abstract — sahip yönetimi fonksiyonları └── is TransactionManager ← abstract — tx yaşam döngüsü + events
OwnerManager.sol ve TransactionManager.sol ayrı deploy edilmez. Bunlar MultiSigWallet'ın içine gömülü. Emine sadece MultiSigWallet ABI'sini kullanır.
Deploy: sadece MultiSigWallet(address[] owners, uint256 numConfirmationsRequired)
ABI: abi/MultiSigWallet.abi.json  ·  Kural: M ≥ N/2  (örn: 3 sahip → en az 2 onay zorunlu)
Constructor
Merve yazdı MultiSigWallet constructor
constructor(
    address[] memory _owners,
    uint256   _numConfirmationsRequired
)
ParametreTipAçıklama
_ownersaddress[] memoryN adet yetkili sahip adresi
_numConfirmationsRequireduint256Gereken min. onay sayısı (M)
Hard-coded kural: M * 2 ≥ N  (M ≥ N/2). Bozulursa revert atar.
Emine kullanır Python deploy örneği
# wallet_service.py içinde
owners = [addr1, addr2, addr3]
M = 2  # 2-of-3

contract = w3.eth.contract(
    abi=MULTISIG_ABI,
    bytecode=MULTISIG_BIN
)
tx = contract.constructor(owners, M)\
    .transact({'from': deployer})
ABI dosyası: abi/MultiSigWallet.abi.json
Core Fonksiyonlar — İşlem Akışı
Emine çağırır submitTransaction
function submitTransaction(
    address _to,
    uint256 _value,
    bytes memory _data
) public
  nonReentrant
  returns (uint256 txIndex)
Sadece sahipler çağırabilir. txIndex döndürür — pending listeye ekler. _data ETH transferinde boş bytes gönderilebilir.
Emine çağırır confirmTransaction
function confirmTransaction(
    uint256 _txIndex
) public
  nonReentrant
  txExists(_txIndex)
  notExecuted(_txIndex)
  notConfirmed(_txIndex)
Onay sayacını artırır. Aynı sahip iki kez onaylayamaz (notConfirmed modifier).
Emine çağırır executeTransaction
function executeTransaction(
    uint256 _txIndex
) public
  nonReentrant
  txExists(_txIndex)
  notExecuted(_txIndex)
Onay sayısı M'e ulaşınca çağrılır. executed = true set edildikten sonra dış çağrı yapılır (CEI pattern).
Emine çağırır revokeConfirmation
function revokeConfirmation(
    uint256 _txIndex
) public
  txExists(_txIndex)
  notExecuted(_txIndex)
  alreadyConfirmed(_txIndex)
Sahip daha önce verdiği onayı geri çekebilir. Yürütülmüş tx'lerde çalışmaz.
Emine okur isConfirmed
function isConfirmed(
    uint256 _txIndex
) public view
  returns (bool)
numConfirmations ≥ numConfirmationsRequired ise true döner.
Emine okur getBalance
function getBalance()
  public view
  returns (uint256)
Cüzdan ETH bakiyesini wei cinsinden döndürür.

Yönetim Fonksiyonları — Multisig Konsensüsüyle Çağrılır
Dikkat: Bu fonksiyonlar dışarıdan doğrudan çağrılamaz. Önce submitTransaction ile teklif oluşturulur, M onay alındıktan sonra executeTransaction ile çağrılır (_data parametresi ABI-encoded fonksiyon çağrısı olur). Emine bu encoding'i contract.encodeABI() ile yapar.
OwnerManager'dan addOwner / removeOwner / replaceOwner
addOwner(address owner)
removeOwner(address owner)
replaceOwner(address owner, address newOwner)
Hepsi onlyWallet modifier'lı. validRequirement modifier M ≥ N/2 kuralını korur.
OwnerManager'dan changeRequirement
changeRequirement(uint256 _required)
Onay eşiğini günceller. Yine onlyWallet + validRequirement zorunlu.
Events — Emine dinler, İrem test eder
Emine dinler İşlem Events
  • SubmitTransaction (txIndex, submitter, to, value, data)
    Yeni pending tx geldiğinde. UI'da listeyi güncelle.
  • ConfirmTransaction (txIndex, confirmer, confirmationCount)
    Onay sayacı artınca. UI'daki sayaç için.
  • RevokeConfirmation (txIndex, revoker)
    Onay geri çekilince.
  • ExecuteTransaction (txIndex, executor)
    TX zincirde yürütülünce. Pending listeden kaldır.
  • Deposit (sender, amount, balance)
    ETH yatırılınca. Bakiye widget'ı için.
Emine dinler Sahip Events
  • OwnerAddition (owner)
    Yeni sahip eklendi. Sahip listesini yenile.
  • OwnerRemoval (owner)
    Sahip çıkarıldı veya değiştirildi.
  • RequirementChange (required)
    M eşiği değişti. UI'daki threshold göstergesini güncelle.
Web3.py event filter örneği:
contract.events.SubmitTransaction.create_filter(fromBlock='latest')
